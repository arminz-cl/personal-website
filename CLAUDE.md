# Project Rules

## Content & Tone
- All text should sound friendly and spoken — not AI-generated or corporate
- Simple, clear English (accessible over fancy; English is a second language)
- Keep it descriptive and visual where possible; people skim, they don't read deeply

## Design
- Simple and clean — avoid complexity for its own sake
- Visual hierarchy matters more than decoration

## Code & Stack
- Minimal stack — only add dependencies if clearly needed
- Framework: Next.js (static export) + Tailwind CSS
- Package manager: npm

## Git
- Descriptive commit messages
- Clean, readable history — one logical change per commit
- No force push, no destructive operations without confirmation

## Deployment
- GitHub Pages via GitHub Actions (static export)
- Repo will be public (open source showcase)
